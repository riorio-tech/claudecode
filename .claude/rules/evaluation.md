# inoue-movie5 動画品質評価ルール

> 適用範囲: `pipeline/eval.js` および生成後の品質判断全般

## 評価エージェントの実行タイミング

- 動画生成が完了したら **必ず自動で評価を実行する**（inpaint.js Step 5）
- 評価をスキップしてはいけない（`--no-assembly` フラグ時のみ例外）

---

## 評価スコア（100点満点・10項目）

| 項目 | 重視ポイント |
|------|------------|
| 商品一致度 | 商品の外観・色・形状・ロゴが元画像と一致しているか（最重要） |
| 商品統一感 | 全フレームで同じ商品が一貫して映っているか |
| 商品差し替え自然さ | 合成の違和感・アーティファクト・歪みがないか |
| モーション品質 | 動きの滑らかさ・ブレのなさ・ジャンプのなさ |
| 映像クオリティ | 解像度・シャープさ・色再現・アーティファクトのなさ |
| 背景照明整合性 | 商品と背景の光源・影・色温度の整合性 |
| TikTok適性 | 縦型構図・視認性・テンポ・購買意欲 |
| 字幕テキスト品質 | 読みやすさ・位置・タイミング・フォントの適切さ |
| 商業的訴求力 | 購買を促進できるか（CVR 観点） |
| フック力 | 冒頭1〜2秒での視聴離脱防止力 |

---

## 判定基準

| スコア | 判定 | 対応 |
|--------|------|------|
| 90〜100 | ★ 優秀 | そのまま使用可 |
| 75〜89  | ◎ 良好 | 使用可（改善提案を参考に） |
| 60〜74  | ○ 普通 | 改善提案を実施してから再生成推奨 |
| 45〜59  | △ 要改善 | 問題箇所を修正して再生成 |
| 0〜44   | ✕ 不良 | 必ず再生成 |

---

## 評価後のアクション

- **改善提案は必ずターミナルに表示し、ユーザーに伝える**
- 総合スコアが 60点未満の場合: 改善点を特定して再生成を提案する
- 特に「商品一致度」が 70点未満の場合は優先的に改善する（商品が違うと売れない）

---

## eval_log.md の管理

- `{outputDir}/eval_log.md` に **追記**（上書き禁止）
- 生成のたびに評価結果が蓄積される → 過去との比較に使う
- Markdown テーブル形式で各項目のスコア・判定・コメントを記録

---

## Claude モデル

評価には `CLAUDE_MODEL`（デフォルト: `claude-sonnet-4-6`）を使用。  
Haiku ではなく Sonnet を使う理由: 10項目 × コメント付きの JSON を正確に生成するため。  
`max_tokens: 3000` 以上を確保すること（低いと JSON が途中で切れる）。
