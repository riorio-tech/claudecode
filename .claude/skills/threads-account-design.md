# Threadsアカウント設計エージェント

このスキルを呼び出したら、以下の手順でThreadsアカウントを設計する。
**パイプライン（run.js）には組み込まない。このスキル経由でのみ実行する。**

## ステップ1: 現状把握

まず以下を読み込む:
- `threads/memory/posted.json`（投稿履歴があれば傾向を把握）
- `threads/docs/superpowers/specs/2026-04-04-threads-agent-design.md`（哲学・ターゲット）

## ステップ2: Webリサーチ

以下の観点でWebSearchを実行する:

```
検索クエリ例:
- "Threads AI初心者 フォロワー増やし方 2024"
- "Threads 伸びるアカウント 特徴 プロフィール"
- "SNS AI教育アカウント 成功事例"
- "Threads bio 書き方 コツ フォロー率"
```

収集する情報:
- フォロワーが伸びるbioの特徴（文字数・構成・言葉遣い）
- AI教育系アカウントで成功しているパターン
- Threadsでバズるプロフィール・世界観の傾向

## ステップ3: アカウント設計

リサーチ結果をもとに、Claude Sonnetでアカウント設計を生成する。

設計内容:
1. **アカウントコンセプト** — 一言でこのアカウントが何者かを定義
2. **ペルソナ** — 投稿者としての人格・キャラクター設定
3. **bio候補3案** — 150文字以内、フォロー率を最大化する構成で
4. **トーン指針** — 言葉遣い・口調・使っていい表現・禁止表現のリスト
5. **ハッシュタグ戦略** — 使うハッシュタグ10選 + 使い分けルール
6. **初期フォロワー獲得戦略** — 最初の1,000人をどう集めるか（具体的なアクション）

## ステップ4: 出力

`threads/docs/account-design.md` に書き出す。
既存ファイルがある場合は**上書きせず**、日付付きセクションとして追記する。

ターミナルに設計サマリーを表示する（bio3案を全文表示）。

## 注意事項

- このスキルはパイプライン（`make POST`）とは完全に独立している
- 実行後にrun.jsを呼び出したり、投稿を行ったりしてはいけない
- ユーザーの確認なしにアカウント設定（bio等）を変更してはいけない
- 設計書の出力のみが責務
