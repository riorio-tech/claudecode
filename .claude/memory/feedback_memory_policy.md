name: memoryに書くべき内容の方針
description: memoryは普遍的な学びのみ記録する。作業進捗・一時状態は書かない
type: feedback
---

memoryに書くのは「次の会話でも活きる普遍的な学び」だけ。

**書くべきもの（AIの成長に繋がる）:**
- 踏んだバグと根本原因・修正パターン
- APIの正しい呼び出し方・ハマりポイント
- ユーザーの好み・作業スタイル・価値観
- 設計判断とその根拠（なぜそうしたか）
- 繰り返し起きる問題のパターン

**書かないもの:**
- 現在の作業進捗・タスク状態
- 「〇〇を実行中」「次のステップは〇〇」
- 特定セッション内でしか意味を持たない情報
- コードの内容（コードはリポジトリが正）

**Why:** memoryはAIを成長させるもの。セッションをまたいで使える知識だけを蓄積することで、次の会話から即戦力になれる。進捗管理はTaskツールやPlanで行う。  
**How to apply:** memory書き込み前に「これは3ヶ月後の別セッションでも役立つか？」を自問する。YESなら書く、NOなら書かない。
