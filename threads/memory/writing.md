# 文章設計ガイド

## 構造パターン（5種類・ランダムで使い分ける）

### パターン1: AtoB（変化の物語）
```
フック（現状Aを1行で）

A（変化前の状態・問題）を描く。

介入・転換点（何が起きたか）

B（変化後の状態・結果）を描く。数字があれば出す。

本質を1行で締める。
```

### パターン2: But-Therefore構文（逆転の論理）
```
フック（一般的に信じられていることを1行で）

But: その前提は間違っている。理由と根拠。

Therefore: だから自分はこうしている。こうすべき。

締め（行動指針を1行で）
```

### パターン3: 問題→解剖→処方（診断型）
```
フック（問題・症状を1行で）

なぜそれが起きるのかを解剖する。構造・原因を掘る。

自分が見てきた具体的な事例。

処方箋を1行で。
```

### パターン4: 体験開示型（ストーリー先行）
```
フック（自分の体験を1行で。感情が入っていい）

その体験の背景と文脈。何をしていたか。

何が起きたか。何を感じたか。

そこから得た判断・気づきを1行で。
```

### パターン5: 数字先出し型（データドリブン）
```
フック（具体的な数字を1行で）

その数字が生まれた文脈・背景。

数字を動かした介入・プロセス。

再現可能な教訓を1行で。
```

## フック設計（冒頭1行・25文字以内）

### フック7型
| 型 | パターン | 例 |
|---|---|---|
| 数字爆弾型 | [数字]という結果が出た。 | 「利用率8%で止まった。」 |
| 断言型 | [短い断言]。 | 「ARRより先に見るものがある。」 |
| 逆説型 | [常識]は間違いだ。 | 「精度を上げるほど売れなくなる。」 |
| 問いかけ型 | [痛い問い]？ | 「そのAI、誰が使ってる？」 |
| 体験開示型 | [自分の体験を1行で] | 「3ヶ月で畳んだ。理由が分かった。」 |
| 宣言型 | [名前をつけた]。 | 「これを『AI墓場』と呼ぶことにした。」 |
| 格差提示型 | [A]と[B]の差は[C]だ。 | 「成功と失敗の分かれ目は最初の問いだ。」 |

## コピーライティングに使える単語・表現

### 好奇心を刺激する
- 誰も言わない / 実は / 正直に言うと / 本音は
- 見落としてる / 気づいてない / 知らない人が多い
- 裏側 / 本当の理由 / 実際のところ

### 緊張感・危機感
- もう遅い / 気づいたら手遅れ / 差がついてる
- 間違ってる / 勘違い / それは幻想
- 負ける / 消える / 置いていかれる

### 信頼・当事者性
- やってみてわかったのは / 現場で気づいた
- 実際に動かして / 自分でやって
- 経験してきた / 見てきた / 体感してる

### 行動・変化
- 変わった / 変えた / 手放した
- 動かした / 実装した / 仕組みを作った
