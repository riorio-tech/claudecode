## 1. システム概要：何をやっているのか

商品画像1枚を入力すると、TikTok Shop用の縦型動画（20〜25秒）を自動生成するパイプライン。

```
商品画像
    ↓
[Step 1] Claude Sonnet  →  10パターンのショットプラン設計（HOOKバリエーション）
    ↓
[Step 2] Sharp + FLUX   →  画角バリエーション画像を100枚生成
    ↓
[Step 3] Seedance Pro   →  静止画 → 5秒動画クリップに変換（100本）
    ↓
[Step 4] ElevenLabs     →  日本語ナレーション音声生成
    ↓
[Step 5] ffmpeg         →  クリップ結合 + 字幕 + カラーグレード → final.mp4
    ↓
[Step 6] Claude Vision  →  12項目×100点の自動品質評価
    ↓
[Step 7] SQLite DB      →  CVR・勝ちパターンを蓄積 → 次回生成の精度向上に利用
```

### 使用AIと役割

| AI | 用途 | 選定理由 |
|----|------|---------|
| Claude Sonnet 4.6 | 台本・ショット設計・品質評価 | 日本語品質・構造化JSON出力の安定性 |
| Claude Haiku 4.5 | 画像品質チェック（軽量タスク） | コスト削減（Sonnetの1/10以下） |
| Seedance 1.5 Pro | 静止画→動画生成 | TikTok系動画に特化・コストバランス |
| FAL FLUX Fill | 商品画像のインペイント | 商品写真の画角バリエーション生成 |
| ElevenLabs v3 | 日本語TTS + タイムスタンプ | 字幕同期に必要なワードレベル時刻取得 |
| Real-ESRGAN | 4Kアップスケール | 1080→4K（TikTok品質基準） |

### 生成コスト・時間

| 項目 | 実績 |
|------|------|
| 生成時間 | 5〜13分/本 |
| API費用 | ¥45〜¥141/本（Seedance回数に依存） |
| 主なコスト | Seedance（1回≒¥15）、ElevenLabs（¥5〜） |

## 2. 品質評価スコアと参考動画との差異

### 評価の仕組み

Claude Vision が動画フレームを解析し、12項目×100点満点で採点する。  
参考動画（目標とする高品質Tumbler動画）を基準として比較評価することが可能。

### 実測スコア（全15回のテストランより）

**参考動画との比較評価**（Tumbler高品質動画を100点基準として相対評価）

| 実行 | 商品 | 総合スコア | 判定 |
|------|------|-----------|------|
| inpaint12 | d'Alba スプレーセラム | **40点** | ✕ 不良 |
| inpaint14 | d'Alba スプレーセラム | **22点** | ✕ 不良 |
| inpaint15 | ネックマッサージャー | **22点** | ✕ 不良 |

### なぜ単体80点なのに比較評価で22点になるのか

参考動画は「bottle bottleタンブラー」の商品動画。  
比較評価では「参考動画と同じ演出スタイル・撮影品質・商品の映り方」を採点する。  
生成動画はd'Albaやネックマッサージャーを正しく映しているが、**参考動画のスタイルでは映せていない**。

> 言い換えると：**「動画として成立している」と「参考動画と同じクオリティ」は全く別の問題**

**判定基準**: 90〜100点 ★優秀 / 75〜89点 ◎良好 / 60〜74点 ○普通 / 45〜59点 △要改善 / **0〜44点 ✕不良**

## 3. ボトルネック分析

### 🔴 Critical（参考動画レベルを阻んでいる根本問題）

#### ① 商品差し替えが機能していない（最重要）

**設計上の想定:**
- Tumblerテンプレート動画を使って、別の商品を同じスタイルで映す
- 参考動画のような「手持ち・接写・底面クローズアップ」を再現

**実際の挙動:**
- Seedanceは入力商品画像をそのまま動画にするが、テンプレートの「撮影スタイル」は引き継がない
- 結果: 参考動画風のスタイルではなく、商品が白背景で静止しているだけの動画が生成される

**参考動画との具体的な差:**

| 演出要素 | 参考動画 | 生成動画 |
|---------|---------|---------|
| 吊り下げシーン | あり | なし |
| 底面クローズアップ | あり | なし |
| 手持ち揺らし | あり（リアル感） | なし（静的） |
| ロゴ接写 | あり | なし |
| カラバリ紹介 | あり | なし |

#### ② フック力の弱さ（CVRに直結）

| | 参考動画 | 生成動画 |
|-|---------|---------|
| 冒頭1秒 | 「ただのタンブラーじゃないんです！」+ インパクト映像 | 商品を手に持つだけ |
| CTA | 「ここから買える↓」「黄色ボタンからチェック」 | 弱い or ない |

冒頭1〜2秒で視聴者の70%が離脱するとされる。フックの差がCVRに直結する。

### 🟡 Major（改善で解決できる問題）

#### ③ 字幕テキストが途中で切れる（技術的バグ）

ElevenLabsのタイムスタンプと字幕レンダリングの連携に不具合。  
例: 「韓国コスメ大注目のd」「Albaファーストスプレーセラム」と分断表示される。

#### ④ カット間の変化が単調

- 現状: ほぼ同じ角度・距離・ポーズの繰り返し
- 参考動画: 正面→底面→俯瞰→接写→カラバリと明確なストーリー構成

#### ⑤ Claude Haiku レートリミット（5req/分）

- 品質チェックが逐次実行 → 10本生成時に時間的ボトルネック

### 🟢 Minor（対応済み）

| 問題 | 対処 |
|------|------|
| Seedanceで商品が2個映る | プロンプト修正（`product already held in hand`を必須化） |
| ffmpeg日本語ファイル名問題 | 出力パスをASCIIに統一 |
| fal.ai SDK のURL構築バグ | raw HTTPで直接 submit/poll/result を実装 |

## 4. コードクオリティ評価

### 設計・構造

| 観点 | 評価 | 内容 |
|------|------|------|
| アーキテクチャ | ★★★★☆ | 9エージェント分離・JSON受け渡し・プロバイダ切替（config.js 1行で全AI切替可能） |
| 可読性 | ★★★★☆ | ESM、Zod検証、カラーlogger を適切に活用。各エージェントが自己完結した async 関数 |
| エラーハンドリング | ★★★☆☆ | `Promise.allSettled` で並列失敗を吸収。API失敗時は最大3回リトライ |
| 自動テスト | ★★☆☆☆ | 単体テストなし。`eval.js` による動画品質チェックが唯一の品質保証 |
| ドキュメント | ★★★★☆ | 各エージェントに `CLAUDE.md` でエスカレーション基準・仕様を明記 |
| セキュリティ | ★★★★☆ | APIキーは `.env` 分離・コードへのハードコードなし |
| **総合** | **★★★☆☆** | 設計は良質。品質保証の自動化が弱点 |

### 特筆すべき設計上の工夫

- **プロバイダ切替**: Runway → Kling → Seedance など、環境変数1行でAIモデルを切替可能。将来のモデル更新に強い
- **DB永続化**: CVR・勝ちパターンをSQLiteに蓄積 → 「使うほど精度が上がる」仕組みの土台
- **VIDEOS_PER_PRODUCT**: 1商品から10本のHOOKバリエーション動画を生成 → TikTok上でのA/Bテストデータを1日で確保

### 現時点の技術的負債

- 自動テストがないため、コード変更時のリグレッション検出ができない
- eval.js は参照動画との比較評価を想定して設計されているが、単体品質評価との使い分けが不明確

## 5. 参考動画との本質的ギャップと今後の対策

### 「このままでは参考動画レベルに到達できない」理由

現在の生成手法（Seedance）は：  
- **できること**: 入力商品画像を忠実に動画化する
- **できないこと**: 参考動画の「撮影スタイル・カメラワーク・演出パターン」を学習して再現する

参考動画の品質の本質は「人が商品を自然に扱う映像」と「強いフック演出」にある。  
これはモデルが変わらない限り、プロンプト調整だけでは再現できない。

### 推奨アクション（優先度順）

| Priority | 施策 | 期待改善 | 難易度 |
|---------|------|---------|--------|
| **P0** | **Kling v2.1 への切り替えテスト** | 商品一致度 +30〜40点 | 低（config.js切替のみ） |
| **P1** | **フックテキスト自動生成の強化**（Claude台本改善） | フック力 +20点 | 中 |
| **P2** | **字幕途切れバグの修正** | 字幕品質 +30点 | 低 |
| **P3** | **カットアングルの多様化**（底面・接写を必須化） | カット間変化 +20点 | 中 |
| P4 | ComfyUI + ControlNet によるオフライン合成の検討 | 商品差し替え精度 大幅向上 | 高 |

### P0: Kling v2.1 切り替えの根拠

Kling v2.1 はSeedanceと比べ「参照画像のアンカリング」が強く、入力画像の商品を動画内で維持する精度が高い。  
config.js の `VIDEO_GEN_PROVIDER` を変更するだけで試験可能（追加実装不要）。  
→ **最短で最大の改善が見込める施策**

### 目標スコアとロードマップ

| フェーズ | 内容 | 目標スコア |
|---------|------|-----------|
| 現在 | Seedance + 現状パイプライン | 30-40点（参照比較） |
| P0〜P2施策後 | Kling切替 + フック改善 + バグ修正 | 50〜65点 |
| Phase 3完了 | 演出・構成の全面改善 | **75点以上**（使用可能水準） |
| Phase 4 | CVR計測ループ稼働 | データによる継続改善 |

## 補足：評価スコアの見方

eval.js が採点する12項目の意味：

| 項目 | なぜ重要か |
|------|----------|
| 商品一致度 | 商品が間違って映ったら売れない。最重要項目 |
| フック力 | 冒頭2秒で視聴者が離れるかが決まる。CVRに直結 |
| 商業的訴求力 | 「買いたい」と思わせるかどうか。最終的なCVRの決定因 |
| TikTok適性 | 縦型・テンポ・視覚的な引き付け。プラットフォーム最適化 |
| 字幕テキスト品質 | 音OFF視聴が多いTikTokでは字幕が主要な情報伝達手段 |
| モーション品質 | ブレ・ジャンプ・不自然な動きは離脱率を上げる |
