# 成果指標 詳細定義

## Layer 2: SNS 先行指標

| 指標 | 定義 | 合格ライン | 改善トリガー |
|---|---|---|---|
| **完視聴率** | 最後まで見た割合 | ≥ 30% | < 20% で フック見直し |
| **CTR（リンククリック率）** | 動画からリンク・プロフへのクリック率 | ≥ 2% | < 1% で CTA 見直し |
| **保存率** | 動画を保存した割合（購買意欲の代理指標） | ≥ 3% | < 1% でスクリプト見直し |
| **フック離脱率** | 冒頭3秒で離脱した割合 | < 50% | ≥ 60% でフック差し替え |

これらが analytics.js で計測する指標。「改善トリガー」を超えたとき、Claude が次回 research.js に警告を渡す。

## Layer 3: システム運用指標

| 指標 | 定義 | 目標値 |
|---|---|---|
| **投稿継続率** | 月の目標投稿数に対する実績割合 | ≥ 90%（月30投稿なら27本以上） |
| **動画生成コスト** | 1本あたりのAPI費用（HeyGen + Claude） | ≤ 200円/本 |
| **担当者工数** | SNS 関連作業に費やす週次工数 | ≤ 2時間/週（導入前比 -80%） |
| **パイプライン成功率** | エラーなく完走したジョブの割合 | ≥ 95% |

## 「成果なし」の判断基準

以下の状態が **4週間継続** した場合、スクリプト戦略を根本から見直す:
- 完視聴率 < 20% かつ フック離脱率 ≥ 60%（コンテンツそのものの問題）
- CTR < 0.5%（プラットフォームとの相性、または商品カテゴリの問題）
- 投稿継続率 < 80%（システム安定性の問題）

## 指標の責任マッピング

```
Layer 1（購入数）    ← ビジネス側が追う。このシステムは貢献するが直接制御しない
Layer 2（SNS指標）   ← analytics.js が計測・Claude が改善提案 → research.js にフィードバック
Layer 3（運用指標）  ← run.js / audit.log / metrics-db.js が追跡
```

## ファン対応（Phase 4 将来対応）

```
pipeline/comment-reply.js  ← 新規 Stage 9（将来対応）

動作:
1. 各プラットフォームのコメントを取得（TikTok/Instagram/YouTube API）
2. Claude Sonnet でコメントを分類（質問・クレーム・絶賛・スパム）
3. カテゴリ別のトーンでAI返信文を生成（ブランドガイドラインに準拠）
4. 返信候補を output/inpaint{N}/comment_replies.md に出力
5. 自動返信 or 人間レビュー後に返信（設定で切り替え）
```

デフォルトは「返信候補の生成のみ」。人間が確認後に投稿する設計。
