# GOAL — SNS自動運用SaaS ゴール定義

> 迷ったときはここに戻る。実装の前にこのファイルを読む。

---

## ゴールの3層構造

```
Layer 3: 大きなビジョン（5〜10年）   ← なぜ作るか
Layer 2: 事業ゴール（1〜2年）        ← 何を達成すれば成功か
Layer 1: 最初の検証ゴール（3〜6ヶ月）← 今何を作るべきか  ← 今ここ
```

---

## Layer 3 — 大きなビジョン

### 「日本のSNSマーケティングの勝ち方を変える」

**現状の問題:**

SNSで売れるかどうかは、予算と人材の量で決まっている。  
大手ブランドは代理店に月100万円払い、プロが動画を作り、A/Bテストを毎週回す。  
中小ブランドはそれができない。資金がなければSNSで戦えない。

**このシステムが完成したとき:**

- 月3万円払えば、大手と同等のSNS運用が自動化される
- データが蓄積されるほど、中小ブランドが大手より「何が効くか」を知っている状態になる
- SNS代行業者が1人で100ブランドを管理できるようになる
- 動画を作ることに時間を使うのではなく、「何を伝えるか」の戦略に集中できる

**一言で言えば: 「SNSマーケティングのAIインフラ」になること。**

予算ではなく、学習データの量と質が競争力を決める世界を作る。

## Layer 2 — 事業ゴール（1〜2年）

### ARR 1,200万円（月100万円）の達成

| プラン | 単価 | 必要社数 |
|---|---|---|
| プロプラン | 月8万円 | 13社 |
| スターター | 月3万円 | 40社 |

**この数字が意味すること:**

「SaaSとして成立している」という証明。  
投資家に見せられる数字。採用できる規模。次の100社に向けて加速できるフェーズ。

**達成の3条件:**

1. **月1社ペースの新規契約** — 最初の6ヶ月は無料/割引で実証を優先
2. **解約率 < 5%/月** — 継続して使われている = 本当に価値がある証拠
3. **セルフサービスで運用** — 人手がなくても顧客が使い続けられる

---

## Layer 1 — 最初の検証ゴール（3〜6ヶ月）← 今ここ

### 「1社目のSNS代行業者がお金を払って使っている」

これが本当の Phase 1 ゴール。  
ここまで到達すれば「ビジネスになる」という確信が持てる。

### 完了条件チェックリスト

- [ ] 1社のSNS代行業者がこのシステムで月30本以上の動画を生成している
- [ ] その業者が月30,000円以上を支払っている
- [ ] その業者の担当者がターミナルを使わずに操作できている
- [ ] 1ヶ月後に更新している（解約していない）
- [ ] 「他の代行業者に紹介したい」と言っている

**全部YESになったとき、Layer 1 は完了。Layer 2 フェーズに移行する。**

## ロードマップ（逆算）

```
[今] CLI ツール（開発者のみ使える）
   ↓ ハーネス Layer A 実装
[M1] 開発者が安心して使えるレベル（クラッシュしても再開できる）

   ↓ Slack Bot Phase 1（スクリプト承認）
[M2] 非エンジニアが Slack だけで動画生成できる

   ↓ SNS自動投稿（Phase 2 配信）
[M3] 商品入稿 → TikTok/Instagram 投稿 が1クリックで完結

   ↓ 最初の1社に無料提供して3ヶ月実証
[M4] 実在するSNS代行業者が実際に使っている ← Layer 1 完了

   ↓ マルチブランド + Web ダッシュボード
[M5] セルフサービスで使える → 課金開始

   ↓ 10社 → 30社 → 100社
[M6] ARR 1,200万円 ← Layer 2 完了
```

## 「ズレ」を防ぐための3つの問い

**実装作業を始める前に、必ずこの問いに答える:**

```
Q1. これは M1〜M4 のどのマイルストーンに向かっているか？
Q2. これがなければ「1社目がお金を払う」は実現しないか？
Q3. これは「データ蓄積」か「オペレーターのスケール」に繋がるか？
```

どれにも当てはまらない作業は、**今やらなくていい可能性が高い。**

---

## 「ゴールが小さくなる」を防ぐための基準

「大きすぎる」と感じる目標が、実は適切なサイズ。

| こう感じたら | 本当はこう考える |
|---|---|
| 「まず自分で使えるようにしよう」 | 顧客に使わせることを先に決める |
| 「まず1機能だけ完成させよう」 | M4（1社目契約）から逆算して必要最小限を決める |
| 「ツールとして完成してから売ろう」 | 未完成でも使わせてフィードバックをもらう |
| 「もう少し磨いてから」 | 磨くより顧客に触れさせる方が速い |
