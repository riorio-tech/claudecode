# 09_eval — 参照動画比較評価エージェント

## 役割

生成した動画を**参照動画（理想の出力）と比較**して品質を 100 点満点で評価する。
評価結果は `eval_log.md` に追記し、改善点をターミナルに表示する。

単独の QA（08）が技術検証なのに対し、
09_eval は「目標とする動画品質に近いか」を主観・定量の両面で評価する。

## 参照動画の設定

参照動画は以下の優先順で使用:
1. `--reference <path>` で明示指定されたファイル
2. `output/reference/` フォルダ内の最高評価動画（`eval_log.md` で追跡）
3. なければ評価スキップ（比較なし・絶対評価のみ）

## 入力

- `{jobDir}/07_assembly/final.mp4` — 評価対象
- `reference_path` — 参照動画パス（任意）
- `01_script.json` — 脚本（テーマ・感情アーク確認用）

## 出力

- `{outputDir}/eval_log.md` への追記
- `{jobDir}/09_eval-report.json`

## 評価スコア（100 点満点・10 項目）

| 項目 | 重視ポイント |
|------|------------|
| フック力 | 冒頭 3 秒でスクロールを止められるか |
| 感情アーク | 感情の山谷が明確に伝わるか |
| 映像クオリティ | シャープさ・色再現・アーティファクトのなさ |
| 人物一貫性 | 全シーンで同じキャラクターが映っているか |
| モーション品質 | 動きの滑らかさ・ブレ・不自然な動きのなさ |
| ドラマ演出力 | 緊張感・対比・沈黙の使い方 |
| 字幕視認性 | 読みやすさ・タイミング・語数の適切さ |
| 音声品質 | 台詞の明瞭さ・BGM バランス |
| クリフハンガー力 | 「次を見たい」と思わせる終わり方か |
| シリーズ継続性 | 前話・次話との連続性（1 話単体なら中立） |

## 判定基準

| スコア | 判定 | 対応 |
|--------|------|------|
| 90〜100 | ★ 優秀 | そのまま投稿可 |
| 75〜89  | ◎ 良好 | 投稿可（改善提案を参考に） |
| 60〜74  | ○ 普通 | 改善後に再生成推奨 |
| 45〜59  | △ 要改善 | 問題箇所を修正して再生成 |
| 0〜44   | ✕ 不良 | 必ず再生成 |

## 参照動画との比較評価（参照あり時のみ）

参照動画と並べて以下を Claude Sonnet（ビジョン）で比較:
- 構図・カメラアングルの質
- 感情表現の強度
- 全体的なドラマ的完成度

比較コメントは日本語で `eval_log.md` に記録。

## eval_log.md フォーマット

```markdown
## {日時} — {jobId} (第{episode}話)

| 項目 | スコア | コメント |
|------|--------|---------|
| フック力 | 8/10 | 冒頭の表情が効果的 |
| 感情アーク | 7/10 | 中盤の沈黙シーンが強い |
...

**総合スコア:** 82/100 ◎ 良好

**改善提案:**
- 第3シーンの台詞を短くする（現在 20 文字 → 12 文字以内推奨）
- BGM が声に被っている（bgmVolume を 0.20 に下げる）

**参照動画との比較:** 参照より構図が良いが、感情のピークが遅い
```

## Claude モデル

`CLAUDE_MODEL`（デフォルト: `claude-sonnet-4-6`）を使用。
ビジョン評価のため Haiku は不可。`max_tokens: 3000` 以上を確保。
